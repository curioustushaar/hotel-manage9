## 2. CHATBOT QUICK OPTIONS

### 2.1 MAKE RESERVATION

**What is a Reservation?**
A reservation is a pre-booking of a hotel room by a guest. Staff can create reservations from the admin panel before the guest arrives.

**How to Make a Reservation (Guest Inquiry Flow):**
1. Contact us at +91 98765 43210 or support@bireenaatithi.com
2. Visit the hotel directly at our Patna office
3. Hotels using Bireena Atithi can book via the admin dashboard

**How Staff Creates a Reservation (Admin Flow):**
1. Login to Admin Dashboard → Go to **Reservations → New Reservation**
2. Fill in guest details: Name, Mobile, Email, ID Proof, Address
3. Select Room Type, Room Number, Check-in Date, Check-out Date
4. Choose Meal Plan, Number of Adults/Children
5. Select Booking Source (Walk-in, Online, Phone, OTA, etc.)
6. Add any Extra Charges or Complimentary Services
7. Choose Payment Mode (Cash, Card, Bank Transfer, UPI)
8. Confirm and save the reservation
9. System generates a unique Booking ID and updates room status to "Booked"

**Reservation Statuses:**
- **Upcoming** – Booking confirmed, guest not yet arrived
- **Checked-in** – Guest has arrived and checked in
- **Checked-out** – Guest has departed
- **Cancelled** – Reservation was cancelled
- **No-show** – Guest did not arrive

**Key Reservation Features:**
- Multi-room booking support
- Early check-in / Late check-out tracking
- Guest document upload (Aadhaar, Passport, etc.)
- Group booking support
- Folio management (billing under one account)
- KOT (Kitchen Order Ticket) linked to reservation

### 2.2 ORDER FOOD

**How Guests Order Food:**
Food can be ordered in two ways:
1. **In-Room Dining:** Guest scans the QR code on the room door → Opens the food order page → Selects items → Places order
2. **Restaurant/Table Service:** Waiter takes order and enters it via Table View → Admin panel KOT

**QR Code Ordering Flow:**
1. Hotel generates Room QR codes (from Property Setup → Generate Room QR)
2. Guest scans the QR code using their phone camera
3. Guest is redirected to the food order page (no login required)
4. Guest selects food items from the menu
5. Order is placed and sent directly to the Kitchen as a KOT
6. Kitchen staff prepares and delivers the food

**Admin Food Order Flow (via Table View):**
1. Go to **Table View** in the sidebar
2. Select the occupied table or room
3. Click "New Order" → Add food items from the menu
4. Submit → KOT is generated and sent to kitchen
5. Order status: Pending → In Progress → Served → Billed

**Food Order Page Features:**
- Category-wise menu display
- Item search
- Add/remove items from cart
- Show price per item
- Quantity adjustment
- Special instructions field
- Order summary before final submit

**KOT (Kitchen Order Ticket):**
- Automatically generated when food is ordered
- Sent directly to kitchen display/printer
- Each item shows: item name, quantity, special notes, table/room number, time
- Kitchen can mark items as "In Progress" or "Served"
- Multiple KOT rounds supported per order

### 2.4 TABLE BOOKING

**What is Table Booking?**
Table booking refers to reserving a specific dining table at the hotel's restaurant for guests dining in.

**How Table Booking Works:**
1. Guest requests a table (via phone/walk-in/QR)
2. Receptionist/Waiter goes to **Table View** in the admin panel
3. Selects the table → Assigns it to the guest
4. Takes the food order → Generates KOT
5. After dining, generates the bill from the Cashier Section

**Table Management:**
- Tables are configured under **Property Configuration → Table Management**
- Each table has: Table Number, Capacity, Status (Available/Occupied/Reserved)
- Tables can be linked to rooms for in-room dining
- Real-time table status visible on Table View dashboard

**Table Statuses:**
- **Available** – Free to seat guests
- **Occupied** – Currently has guests seated
- **Reserved** – Pre-booked for a guest at a specific time

**Waiter Flow:**
1. Login with Waiter credentials
2. Navigate to Table View
3. Select available table → Mark as Occupied
4. Take order → Enter in system → KOT sent to kitchen
5. When guest is done → Process bill via Cashier

- **URL:** `/secure-owner-login` or `/superadmin/login`

- **After Login:** Redirected to `/super-admin/dashboard`

### Staff Login
- Same login page as Admin
- After login, sees only modules permitted by their Admin

### Default Route After Login (Priority Order)
1. Super Admin Dashboard (if Super Admin)
2. Dashboard (if Admin/Manager/Receptionist/Accountant/Staff)
3. Table View (if Waiter with no dashboard)

### Protected Routes
- All `/admin/*` routes require authentication
- All `/super-admin/*` routes require Super Admin role
- Unauthenticated users are redirected to login
- Wrong-role users see "Access Denied"

## 8. CASHIER SECTION

**Route:** `/admin/cashier-section`
**Access:** Admin/Manager/Accountant (Full), Receptionist (View/Create)

### Purpose
The Cashier Section is where payment transactions are managed and processed. It is the financial hub of the hotel.

### What You Can Do Here
- View all pending payments / outstanding balances
- Process payments (Cash, Card, UPI, Bank)
- Issue receipts/invoices
- Apply discounts to bills
- Split bills between multiple guests
- Transfer charges between folios
- View payment history for any guest
- Handle cancellation refunds
- Print/email receipts

### Cashier Workflow
1. Guest comes to counter for check-out
2. Cashier opens the guest's folio
3. Reviews all charges: Room, F&B, Extras, Taxes
4. Clicks "Add Payment" → Selects mode → Enters amount
5. System marks balance as Paid
6. Invoice auto-generated
7. Can print or email the invoice

### Payment Modes Supported
- **Cash**
- **Credit/Debit Card (Swipe)**
- **UPI / QR Code**
- **Bank Transfer / NEFT / RTGS**
- **Di Card** (Dine-In Card)

## 19. HOUSEKEEPING VIEW

**Route:** Reservations → Housekeeping View
**Access:** Admin/Manager (Full), Staff with "HouseKeeping View" permission

### Purpose
Track and manage room cleaning and maintenance tasks.

### Housekeeping Dashboard
- List of rooms with their housekeeping status:
  - **Clean** – Room is clean and ready
  - **Dirty** – Needs cleaning (post check-out or guest request)
  - **In Progress** – Being cleaned right now
  - **Inspected** – Cleaned and verified by supervisor
  - **Out of Order** – Under maintenance

### Housekeeping Task Flow
1. Guest checks out → Room status becomes "Dirty"
2. Housekeeping supervisor sees it on dashboard
3. Assigns task to a room attendant
4. Attendant cleans the room → Marks as "In Progress"
5. Attendant finishes → Marks as "Inspected"
6. Supervisor verifies → Marks as "Clean"
7. Room becomes "Available" for new bookings

### Features
- Filter rooms by floor, status, room type
- Assign specific staff to housekeeping tasks
- Add notes per room (e.g., "Guest requested extra towels")
- Track time of cleaning start and completion
- Housekeeping schedule management

## 20. ROOM SERVICE

**Route:** Reservations → Room Service
**Access:** Admin/Manager (Full), Staff with "Room Service" permission

### Purpose
Manage in-room food and service requests from guests who are checked in.

### Room Service Flow
1. Guest calls reception or scans room QR to order
2. Staff receives order in Room Service dashboard
3. Order forwarded to kitchen as KOT
4. Kitchen prepares and delivers to room
5. Charges automatically added to guest's folio
6. Guest pays at checkout or pays immediately

### Room Service Dashboard Shows
- Active room service orders
- Order details: Room number, items, quantity, time placed
- Status: Received / In Kitchen / Out for Delivery / Delivered
- Staff can update status in real-time

### How Subscriptions Work
